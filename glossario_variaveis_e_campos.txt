Glossário de variaveis e campos usados no sistema

Aplicativo Atividades:

    Models:

        Instituicao:

            fixo_mensal_inst = Campo usado para definir de a instituição trabalha com pagamento em valor fixo ou mensal;

            cod_fixo = Campo que detém o código da intituição para o caso de fixo Mensal. Quando uma atividade  cadastrada para essa instituição for tickada "fixo_mensal_ativ", ela recebe esse código em seu campo "cod_fixo_ativ". Assim o sistema saberá que essa atividade faz parte do fixo que é pago por aquela instituição;

        Atividades:

            cod = Campo dedicado a unificar atividades da mesma sequência. Ele é usado para controle na lógica de deleção e edição de atividades criadas em sequencia, dando ao usuario a oportunidade de realiza-las solo ou em toda a sequencia. 

            id_vir = Esse campo serve para marcar as atividades que precisam ser quebradas quando a virada de dia (O período atravessa a meia noite). Ele é usado para que o sistema sempre trate sistema com o mesmo id_vir como iguais.

            fixo_mensal_ativ = Campo que indica que uma atividade deve ser atribuida como parte do valor fixo de determinada instituição.

            cod_fixo_ativ = Esse campo recebe o valor do campo "cod_fixo" da instituição onde ocorrerá a atividade. Assim o sistema entende que ele faz parte do hall de atividades pagas pelo valor fixo da unidade.

    Forms:

        AtiviadesForms:

            extra_param = Parametro extra enviado para que o sistema saiba qual foi a escolha do usuário no momento de deletar ou editar atividades que podem ser modificadas solo ou em sequencia.
